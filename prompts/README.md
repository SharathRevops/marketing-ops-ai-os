# Prompts

Production prompts used inside the systems documented in `/case-studies`, not generic prompt examples.

Each file should include:

```
## Used in
[link to the case study that uses this prompt]

## Purpose
[one line — what decision or output this prompt produces]

## Prompt
[the actual prompt, with {{variables}} marked]

## Notes
[anything you learned tuning this — what didn't work, what changed]
```

Keeping the "Notes" section honest (what failed, what you iterated on) is more credible to a technical reviewer than a polished prompt with no history.
